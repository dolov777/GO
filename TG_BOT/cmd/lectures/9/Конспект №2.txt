Давайте рассмотрим типовые алгоритмы обработки массивов и основные термины.

Основные термины:

*   Массив: Структура данных, хранящая упорядоченный набор элементов одного типа. Доступ к элементам массива осуществляется по их индексу (номеру позиции).
*   Индекс: Целое число, указывающее позицию элемента в массиве. Обычно индексация начинается с 0.
*   Длина массива: Количество элементов, хранящихся в массиве.
*   Сортировка: Упорядочивание элементов массива по определенному критерию (например, по возрастанию или убыванию).
*   Поиск: Нахождение элемента с заданным значением в массиве.
*   Фильтрация: Создание нового массива, содержащего только элементы, удовлетворяющие определенному условию.
*   Преобразование (маппинг): Применение заданной функции к каждому элементу массива и создание нового массива с результатами.

Типовые алгоритмы:

1. Поиск:

    *   Линейный поиск: Последовательная проверка каждого элемента массива, пока не будет найден нужный элемент или пока не будет достигнут конец массива. Простой, но неэффективный для больших массивов.
    *   Бинарный поиск: Работает только с отсортированными массивами. Делит массив пополам и сравнивает искомый элемент со средним. Если элемент меньше среднего, поиск продолжается в левой половине, иначе — в правой. Значительно быстрее линейного поиска.

2. Сортировка:

    *   Сортировка пузырьком (Bubble Sort): Многократное прохождение по массиву, сравнение соседних элементов и их обмен местами, если они находятся в неправильном порядке. Простой, но неэффективный для больших массивов.
    *   Сортировка вставками (Insertion Sort): Берет каждый элемент и вставляет его в правильную позицию в уже отсортированной части массива. Эффективен для небольших массивов или почти отсортированных массивов.
    *   Сортировка выбором (Selection Sort): Находит минимальный элемент в неотсортированной части массива и меняет его местами с первым элементом неотсортированной части. Простой, но не очень эффективный.
    *   Сортировка слиянием (Merge Sort): Рекурсивно делит массив на две половины, сортирует каждую половину, а затем объединяет отсортированные половины. Эффективный и стабильный алгоритм.
    *   Быстрая сортировка (Quick Sort): Выбирает опорный элемент, разделяет массив на элементы меньше и больше опорного, а затем рекурсивно сортирует каждую часть. В среднем очень быстрый, но может быть неэффективным в худшем случае.

3. Обход (итерация):

    *   Последовательный обход: Перебор элементов массива один за другим, от начала до конца. Самый простой способ обхода.

4. Преобразование (маппинг):

    *   Применение функции к каждому элементу массива и создание нового массива с результатами. Например, умножение каждого элемента на 2.

5. Фильтрация:

    *   Создание нового массива, содержащего только элементы, удовлетворяющие определенному условию. Например, выбор только четных чисел.

6. Агрегация:

    *   Вычисление некоторого значения на основе всех элементов массива. Например, сумма всех элементов, среднее значение, максимальный или минимальный элемент.

7. Другие операции:

    *   Вставка: Добавление нового элемента в массив.
    *   Удаление: Удаление элемента из массива.
    *   Копирование: Создание копии массива.
    *   Поворот: Сдвиг элементов массива влево или вправо.
